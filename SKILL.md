# 嵌入式秋招雷达

## 项目定位

为嵌入式校招学生打造的全自动投递进度看板 + 岗位发现雷达。核心能力：

1. **飞书多维表格**作为数据底座，主表记录投递进度，机会发现池记录新开放岗位
2. **WebSearch 自动搜索**已开放 2027 届（2026 年秋季招聘）的嵌入式相关校招/提前批岗位
3. **Playwright 动态渲染**抓取招聘页面具体岗位名称、JD、投递链接
4. **Flask 本地大屏看板**实时展示投递进展、机会池、统计数据，带截止时间提醒
5. **定时扫描**每 N 小时自动运行，发现新机会写入飞书并推送提醒

## 核心产品逻辑

```text
机会发现池（每日自动发现的新开放公司/岗位）
  ↓ 用户确认要投
主投递表（用户真实投递过的公司记录）
```

关键区分：
- **机会发现池**：放所有检测到的 2027 届校招机会，带【公司校招开放】或【嵌入式岗位开放】分类
- **主投递表**：只放你确认要投/已投的真实记录，手动维护投递进展

### 隐式校招的发现方法

有些公司不会明确写「2027届校招」字样，但确实面向 2027 届同学。识别方法：

```text
1. 扩大搜索词：搜"2027届"找不到时，改搜"校园招聘""优才计划""精英计划""领军人才"
2. 看 JD 毕业时间：2026年9月-2027年12月毕业 → 就是 2027 届
3. 看公司招聘官网：有"校园招聘"/"应届生"入口且岗位匹配 → 大概率包含 2027 届
4. 关注高校就业网：清华/北大/浙大/上交/西交/华科等就业网的校招公告，经常有低调公司
5. 关注明星创业公司：机器人/具身智能/芯片/AI 方向的独角兽，校招窗口短但岗位含金量高
6. 看启动时间：6-8 月启动的校招项目，大概率就是面向 2027 届

**全覆盖搜索策略**（每轮扫描至少跑 8 组搜索，防止遗漏）：
```text
1. "2027届 嵌入式 校招"
2. "2027届 嵌入式 提前批 秋招 7月 8月"
3. "2027届 嵌入式 校园招聘 汇总"
4. "2027届 校招 嵌入式 BSP RTOS MCU 芯片 机器人 无人机 电源"
5. site:zhiye.com "2027" 嵌入式
6. site:jobs.feishu.cn "2027" "嵌入式"
7. "2027届 秋招提前批 汇总"（第三方汇总：牛客/知乎/秋招日报）
8. 针对已覆盖列表，逐家搜"<公司名> 2027届 校招"验证官方链接
```

**示例**：智元机器人「优才计划」标题没写"2027届"，但 JD 注明 2026.9-2027.12 毕业 → 确认是 2027 届正式校招。

### 筛选标准（入库前必须验证）

写入机会发现池必须同时满足：

```text
1. 是 2026 年开放的校园招聘（2027届）
2. 面向 2027 届同学
3. 有具体公司名 + 来源链接
4. 与嵌入式方向相关（芯片/驱动/RTOS/Linux/MCU/车载/BSP）
5. 排除：2026届/2025届旧信息、培训广告、纯实习无关内容
6. 必须是正式校招（提前批/秋招/春招），排除：实习、暑期实习、实习转正、
   校储实习生、暑期实践等非正式校招项目
7. 信息必须验证属实，不能胡编乱造。没有 2027 届校招的公司不要强行写入
```

### 投递链接来源优先级

```text
1. 优先：公司官方校招官网（*.zhiye.com、公司域名/campus 等）
   前提：官网确实有 2027 届校招内容，经过验证
2. 其次：公司官方招聘公众号/飞书招聘页
3. 不用：高校就业网、mailto 邮箱、第三方招聘平台（应届生求职网/牛客网等）
   → 这些仅作为"发现来源"记录在来源链接中，不作为投递链接
```

**校验 + 链接选择的完整逻辑**：
1. 搜索验证该公司是否发布了 2027 届校招。注意：有些公司不写"2027届"但通过"优才计划""精英计划"或 JD 毕业时间（2026.9-2027.12）间接标明 → 参考「隐式校招的发现方法」
2. 有官网 → 验证官网是否有 2027 届校招内容
   - 有 → 用官网链接作为投递链接
   - 没有 → 用高校就业网/第三方上已验证的岗位链接作为投递链接
3. 没有官网（小公司/初创）→ 搜索引擎找到的岗位来源链接直接作为投递链接
4. 发现渠道：WebSearch 搜"2027届 嵌入式 校招" + 各大高校就业网 + 公司招聘官网 + 行业媒体
5. 同步字段到主表时，也要确保「公司规模」已填写（根据搜索结果补充，如"1000-1500人""100-499人"等）
6. 新增公司到主表时，必须同时填写以下全部字段，不能留空：
   - 公司名称（必填）
   - 公司/行业类型（必填）— 从主表已有选项中匹配，选最合适的 1-2 个
   - 细分类型（必填）— 更细粒度分类，从主表已有选项中匹配
   - 嵌入式方向（必填）— 从主表已有选项中选择，基于 JD 关键词
   - 工作地点（必填）— 基于 JD 或公司总部
   - 公司规模（必填）— 根据搜索信息填写，如"500-1000人"
   - 公司简介（必填）— 一句话概括业务 + 嵌入式方向
   - 岗位类型（必填）— 提前批/秋招/春招，基于 JD 判断
7. 写飞书 API 时用中文字段名（不用字段 ID），避免 ID 变更导致失败
8. 池子已有选项（公司/行业类型、细分类型、嵌入式方向等）不要随意新增，优先从现有选项中匹配

| # | 字段名 | 填写来源 | 示例 |
|---|---|---|---|
| 1 | 公司名称 | 搜索验证 | "台达" |
| 2 | 公司/行业类型 | 从主表已有选项中匹配 | "外企" |
| 3 | 细分类型 | 从主表已有选项中匹配 | "新能源" |

- 公司简介保持一句话，控制在 50 字以内

### 分档写入规则

**A档：嵌入式岗位开放**
- 有具体岗位名称
- 岗位与嵌入式相关（命中嵌入式关键词）
- 有 JD/岗位描述
- 有投递链接/详情链接

**B档：公司校招开放**
- 明确开放 2027 届校招
- 有官方投递入口
- 但未解析到具体嵌入式岗位名

**不入库**：仅有"校招入口可访问"但无 2027/27 届明确信息

### 岗位分类规则

写入主表时必须准确标注「岗位类型」，基于 JD 原文逐条判断，不要凭公司知名度乱填：

```text
判断标准：
- JD 明确写"提前批" → 提前批
- JD 写"秋招"/"秋季校园招聘"/"正式校招"但未提提前批 → 秋招
- JD 写"春招" → 春招
- 时间在 6-7 月启动但不叫"提前批"的，如大疆「拓疆者」是整个秋招的唯一项目 → 秋招
- "未来领军""顶尖人才""精英计划"等特殊人才项目，时间大幅前置且独立于正式批 → 仍按 JD 实际标注来
```

**反例（已踩过的坑）**：
- 大疆「拓疆者」：6月启动、时间早，但这是大疆唯一的秋招项目 ≠ 提前批 → 应标「秋招」
- 小米「全球顶尖人才」：第三方平台标注为"提前批"，面向 2024-2027 届 → 应标「提前批」
- 锐明「星火计划」：JD 写的是暑期实习生 → 不入库

## 关键注意事项

### 飞书 API 反模式（必读）

1. **中文字段名 vs 字段 ID**：飞书多维表格字段被删改后字段 ID 可能失效。**优先使用中文字段名**，写入文件时用 Write 工具生成 UTF-8 JSON，再用 curl `--data @文件` 方式发送，**绝不要在终端内联中文**（Windows Shell 编码会破坏 UTF-8）。

2. **多维表格 URL 类型字段**：格式为 `{"link":"url","text":"描述"}`，text字段为纯文本不传 link 对象。

3. **每次写入前先读字段列表**：确认目标字段名和 record_id 没有因用户手动编辑而改变。

4. **batc_update 单次最多 500 条**。

5. **视图字段顺序**：飞书 API 的 grid 视图不支持通过 API 持久化字段顺序（`property.field_order` 传了就丢失）。用户需要手动拖动。

### 岗位发现原则

1. **不要胡编乱造**：所有写入机会池的岗位必须能追溯到具体来源链接
2. **分清楚实习和秋招**：实习（暑期实习、转正实习）和正式提前批/秋招要分开，后者标"实习可转正"的仅供参考
3. **岗位名不能是 JD 职责句**：例如 `4.负责飞控系统各类传感器驱动和外设接口软件开发` 只能放进 JD，不可当作岗位名称
4. **语义去重**：同公司 + 同发现类型 + 同投递/来源入口视为同一机会簇，只保留质量最高记录
5. **公司级冗余清理**：同一公司已有「嵌入式岗位开放」时，删除低质量「公司校招开放」记录
6. **时效性**：搜索时明确限定"2027届 / 2026年校园招聘"，不要纳入旧信息
7. **如实标注可信度**：官网来源→高；高校就业网→中高；第三方平台→中

### 机会池→主表同步规则（sync_apply_url_deadline.py）

每次扫描后或用户手动触发时，执行机会发现池到主表（秋招统计表）的字段同步。

**同步字段**：
- 投递链接（URL类型，必须传 `{"link":"url","text":"url"}` 对象）
- 投递截止时间（文本类型）
- 秋招岗位（文本类型）

**同步规则**：

1. **只同步「岗位开放状态=已开放」的公司**：其他状态（疑似开放/待确认/已截止）的主表对应字段保持为空，不覆盖
2. **官方链接优先**：投递链接必须用官方校招入口，不能用高校就业网、mailto、第三方招聘平台的链接。池子里存的是非官方链接的，写入主表时自动替换为官方链接
3. **已知官方链接修正表**（持续维护）：
   - 锐明技术 → `https://streamax.zhiye.com/campus`
   - 比特大陆 → `https://jobs.bitmain.com.cn`
   - 中国航发涡轮院 → `https://zhaopin.aecc.cn`
   - 中船凌久电子 → `http://www.csic-lincom.cn`
4. **公司名匹配**：池子「疑似公司」匹配主表「公司名称」，大小写不敏感。已配置别名映射：乐鑫科技↔乐鑫（主表名）
5. **幂等安全**：可重复执行；同步前必须按主表公司聚合机会池记录，每个主表 `record_id` 最多生成一次 update
6. **不允许低质量覆盖**：空截止时间不能覆盖非空截止时间，职责句不能覆盖正常岗位名，`待确认具体岗位` 不能覆盖具体岗位
7. **用中文字段名**（不用字段 ID），避免 ID 变更导致失败

```bash
# 手动执行同步
py src/sync_apply_url_deadline.py

# 审计完整性

## 典型工作流程

### 首次搭建（拿到手 10 分钟跑通）

#### 第 1 步：复制飞书多维表格模板到你自己空间

飞书多维表格模板：https://j0pbq4vb3lh.feishu.cn/wiki/Niv3we4Ldiw56LkEWV2cLuCynvc

打开链接 → 申请**阅读权限**（我会通过）→ 点击右上角「...」→ **复制此表格**到你自己的飞书空间。复制后就得到了你自己的副本，里面有建好的两张子表和全部字段选项。

**⚠️ 重要：复制完成后，打开你副本的链接，记下 `base/` 后面的那串 ID。** 

比如你的副本链接是：
```
https://j0pbq4vb3lh.feishu.cn/base/LiXxXxXxX?table=tblXXXX
```
那 `LiXxXxXxX` 就是你的 `FEISHU_APP_TOKEN`，马上第 3 步要用。

#### 第 2 步：创建飞书自建应用
1. 打开 https://open.feishu.cn/app → 创建企业自建应用
2. 左侧菜单 → 权限管理 → 搜索"多维表格" → 开通：
   - `bitable:app`（多维表格）
3. 左侧菜单 → 安全设置 → 添加 IP 白名单（填入你的本机公网 IP，或先跳过）
4. 左侧菜单 → 凭证与基础信息 → 复制 **App ID** 和 **App Secret**
5. 左侧菜单 → 版本管理与发布 → 创建版本并发布（上线后应用才生效）

#### 第 3 步：填写 .env
将项目目录下的 `.env.example` 复制为 `.env`，填入：
```bash
FEISHU_APP_ID=cli_xxxxxxxx       # 第 2 步复制的 App ID
FEISHU_APP_SECRET=xxxxxxxxxxxx   # 第 2 步复制的 App Secret
FEISHU_APP_TOKEN=XXXXXXXXX       # 第 1 步复制表格后 URL 中的那串
MAIN_TABLE_ID=tblxxxxxxxx        # 模板中的主投递表 ID（复制表格后不变）
DISCOVERY_TABLE_ID=tblxxxxxxxx   # 模板中的机会发现池 ID（复制表格后不变）
```
> 模板复制后 TABLE_ID 通常不变。如果变了，打开飞书表格 → 点击子表名旁的「...」→ 复制链接，URL 中 `table=` 后面的就是 TABLE_ID。

#### 第 4 步：安装依赖
```bash
cd embedded-job-radar
py -m pip install -r requirements.txt
py -m playwright install chromium   # 首次约 2 分钟，后续不需要
```

#### 第 5 步：录入第一批监控公司
打开飞书多维表格 → 进入「嵌入式秋招主投递表」→ 手动填写第一批公司（至少填写：**公司名称、投递链接**）。

> 内置 20 家种子公司模板在 `data/seed_companies.json`，可以手动对照录入。

#### 第 6 步：启动看板
```bash
py src/app.py
```
打开 `http://localhost:8765` → 首次扫描自动执行 → 看板展示。

> 如果主表为空，看板会打印提示引导你录入公司，不会报错。

### 每日维护

1. 打开看板查看今日新增机会
2. 对高匹配岗位进行人工确认
3. 在主表填写投递进展（岗位类型、进展、意愿）
4. 看板每 60 秒自动刷新

### 新增公司适配

每增加一家公司作为监控目标：
1. 搜索该公司招聘官网/校招入口
2. 验证页面可访问性
3. 填入主表：公司名称、公司/行业类型、细分类型、嵌入式方向、投递链接
4. 下次扫描时自动检测该公司开放状态
5. 如果是主流招聘系统（北森/moka/自主），适配 Playwright 抓取器
